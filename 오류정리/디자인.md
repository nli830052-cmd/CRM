# 2026-03-21 디자인 및 AI 기능 고도화 정리

## 문제 1: AI 분석 시 "404 gemini-1.5-flash not found" 에러 발생
- **문제**: 녹음 파일 업로드 후 AI 분석 단계에서 모델을 찾을 수 없다는 404 오류 발생.
- **원인**: Google API의 특정 버전에서 `gemini-1.5-flash` 단축 명칭이 인식되지 않거나 버전 미스매치 발생.
- **해결**: `ai_service.py`의 모델 식별자를 `gemini-1.5-flash-latest`로 변경하여 최신 엔드포인트와 강제로 연결함.

## 문제 2: 웹 대시보드 디자인의 시인성 및 프리미엄 느낌 부족
- **문제**: 기존 UI가 단순 텍스트 위주로 1.3만 건의 대용량 데이터를 관리하기에 부적합하고 투박함.
- **해결**: `static/index.html` 전체 개편.
    - **테마**: Deep Dark (HSL 230 기반) 프리미엄 다크 모드 적용.
    - **디자인**: 유리 질감(Glassmorphism) 및 카드 레이아웃 도입.
    - **대시보드 홈**: 전체 고객 수, 분석된 녹음 파일 수 등 주요 통계를 첫 화면에 배치.
    - **인텔리전스 뷰**: 고객 클릭 시 Gemini가 분석한 영업 인사이트(추천 액션 등)를 카드 형태로 강조.

## 문제 3: 데이터 동기화 확인의 어려움 (1.3만 건 주소록)
- **문제**: 안드로이드에서 보낸 대량의 주소록 데이터가 실제로 DB에 다 들어왔는지 사용자가 확인할 방법이 없음.
- **해결**: 
    - 서버(`main.py`)에 `/stats/` 전역 통계 엔드포인트 신설.
    - 웹 UI 메인 상단에 '전체 고객 수' 카운터를 실시간 표시하여 동기화 상태를 직관적으로 확인 가능하게 함.
